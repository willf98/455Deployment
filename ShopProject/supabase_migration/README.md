# Supabase Migration: ShopProject

Migrates `shop.db` (SQLite) → Supabase (PostgreSQL).

## Files in this folder

| File | Purpose |
|------|---------|
| `01_schema.sql` | Creates all 7 tables with proper PostgreSQL types |
| `02_reset_sequences.sql` | Fixes auto-increment sequences after import |
| `csv/customers.csv` | 250 rows |
| `csv/products.csv` | 100 rows |
| `csv/orders.csv` | 5,001 rows |
| `csv/order_items.csv` | 15,023 rows |
| `csv/shipments.csv` | 5,000 rows |
| `csv/product_reviews.csv` | 3,000 rows |
| `csv/order_predictions.csv` | 1 row |

---

## Step-by-Step Import Guide

### Step 1 — Create the schema

1. Go to your Supabase project → **SQL Editor**
2. Click **New query**
3. Paste the entire contents of `01_schema.sql`
4. Click **Run** (▶)

You should see `Success. No rows returned` for each statement.

---

### Step 2 — Import CSV data (in dependency order)

Import tables in this exact order so foreign keys resolve correctly:

1. `customers`
2. `products`
3. `orders`
4. `order_items`
5. `shipments`
6. `product_reviews`
7. `order_predictions`

**For each table:**

1. Go to **Table Editor** in the left sidebar
2. Click the table name
3. Click **Insert → Import data from CSV** (top right)
4. Upload the matching CSV from the `csv/` folder
5. Confirm column mapping looks correct, then click **Import**

> **Tip:** If you prefer the command line, you can use `psql` with the connection string from Supabase → Project Settings → Database:
> ```bash
> psql "postgresql://postgres:[PASSWORD]@[HOST]:5432/postgres" \
>   -c "\COPY customers FROM 'csv/customers.csv' CSV HEADER"
> ```
> Repeat for each table in the order above.

---

### Step 3 — Reset auto-increment sequences

After importing all CSVs, the identity sequences are still at 1. Run this to fix them so new inserts don't collide with existing IDs.

1. Go back to **SQL Editor → New query**
2. Paste the contents of `02_reset_sequences.sql`
3. Click **Run**

---

Run these quick checks in the SQL Editor:

## Type Changes (SQLite → PostgreSQL)

| SQLite | PostgreSQL | Why |
|--------|-----------|-----|
| `INTEGER PRIMARY KEY AUTOINCREMENT` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | Standard PostgreSQL identity |
| `INTEGER` (0/1 booleans) | `BOOLEAN` | Native boolean; CSVs use `true`/`false` |
| `TEXT` (date only) | `DATE` | `birthdate` |
| `TEXT` (datetime) | `TIMESTAMPTZ` | All datetime fields |
| `REAL` (money) | `NUMERIC(10,2)` | Avoids floating-point rounding |
| `REAL` (risk_score) | `NUMERIC(5,2)` | 0–100 range |
